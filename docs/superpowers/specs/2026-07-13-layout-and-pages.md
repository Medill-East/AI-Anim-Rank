# AI Anim Rank：版式优化与中性公开地址设计

## 目标

让 Top 300 榜单在桌面端更易扫读、个人标记可直接操作且不产生突兀换行；同时使用不包含个人账户名称的免费 Cloudflare Workers 子域名公开访问。

## 保持不变

- 排名数据、排序和筛选语义不变。
- 个人进度仍默认只存于当前浏览器；备份、恢复短语和端到端加密同步的安全边界不变。
- 浅色 / 深色主题、全页连续滚动、移动端卡片视图继续保留。

## 桌面版信息层级

1. 页首保留品牌、主题切换和一句说明，但压缩其垂直空间。
2. 「我的进度」保留为榜单前的快速摘要。备份与私密同步移入更克制的工具区，默认不抢占主工作区。
3. 在筛选区之前增加可展开的「排名依据」说明，简要列出数据来源、加权方式、标准化原则、名称匹配和数据版本；说明明确这是可复核的静态汇总，不是实时评分或唯一权威答案。
4. 筛选区的每个字段都使用一个独立的「标签 + 控件」单元；搜索字段较宽，其余字段等宽，重置按钮与字段底部对齐。
5. 榜单采用稳定列宽：排名、年份和评分为窄列；标题为主列；类型为可读的中列；个人标记为固定宽度操作列。

## 排名依据说明内容

- 原始数据源：MyAnimeList、AniList、Bangumi；页面写明每个平台代表不同的社区样本，不声称其结果完全等价。
- 得分：各源先统一换算为 0–100；三个来源等权取算术平均。样本量只用于最低门槛筛选，不参与当前综合分的权重；展示当前数据的方法学版本，而非编造未入库的实时权重或分数。
- 合并：作品以可审阅的跨站点映射合并，续作、剧场版和独立作品保持独立条目，避免季与季之间重复计入。
- 局限：不同站点更新速度、收录范围、地区语言和评分分布不同；排名适合作为发现作品的入口，不能替代个人判断。

## 长文本和标记规则

- 中文标题最多两行；原名仅显示一行，溢出时省略，而不是拆成零碎多行。
- 类型最多两行，避免挤压作品标题和个人标记列。
- 四个个人标记改为固定的 2 × 2 按钮组。每个按钮文字保持单行，状态颜色与现有语义一致。
- 点击行仍打开详情；点击标记按钮仍只更新对应私有记录，且不打开详情。

## 响应式边界

- 宽屏：上述稳定表格布局，表头继续 sticky。
- 中等宽度：筛选字段自动分行，但字段内部不拆开；表格保持列规则，必要时允许横向查看而不压缩成不可读的断词。
- 小屏：继续使用现有卡片列表；个人标记沿用 2 × 2 结构并与卡片标题对齐。

## 中性公开地址

- 当前页面品牌 `AI Anim Rank` 不包含个人名；个人名只出现在 Workers 账户子域名中。
- 使用 Cloudflare Dashboard 中可配置的账户 `workers.dev` 子域名；部署地址采用 `ai-anim-rank.play-with-experiences.workers.dev`，不暴露个人账户名。
- 现有远程同步尚未配置，因此该部署不改变已启用功能。将来若启用同步 API，可继续由独立 Worker 承载；页面只需更新其 API 配置，不需要将个人进度公开。

## 验收标准

- 1440px 宽桌面下，筛选字段的标签与控件不再错位，四个个人标记呈整齐 2 × 2。
- 常见长标题和长类型不会造成评分、标记列的非预期换行。
- 默认 Light 与 Dark 均可读，连续浏览 300 条记录不受内部滚动容器限制。
- Workers 地址可公开访问，且地址中不包含用户个人名。
- 既有进度联动、备份导入导出、筛选排序、PWA 与加密同步相关测试继续通过。
