# AI Anim Rank：版式优化与中性公开地址设计

## 目标

让 Top 300 榜单在桌面端更易扫读、个人标记可直接操作且不产生突兀换行；同时将公开访问地址迁至不包含个人账户名称的免费 Cloudflare Pages 域名。

## 保持不变

- 排名数据、排序和筛选语义不变。
- 个人进度仍默认只存于当前浏览器；备份、恢复短语和端到端加密同步的安全边界不变。
- 浅色 / 深色主题、全页连续滚动、移动端卡片视图继续保留。

## 桌面版信息层级

1. 页首保留品牌、主题切换和一句说明，但压缩其垂直空间。
2. 「我的进度」保留为榜单前的快速摘要。备份与私密同步移入更克制的工具区，默认不抢占主工作区。
3. 筛选区的每个字段都使用一个独立的「标签 + 控件」单元；搜索字段较宽，其余字段等宽，重置按钮与字段底部对齐。
4. 榜单采用稳定列宽：排名、年份和评分为窄列；标题为主列；类型为可读的中列；个人标记为固定宽度操作列。

## 长文本和标记规则

- 中文标题最多两行；原名仅显示一行，溢出时省略，而不是拆成零碎多行。
- 类型最多两行，避免挤压作品标题和个人标记列。
- 四个个人标记改为固定的 2 × 2 按钮组。每个按钮文字保持单行，状态颜色与现有语义一致。
- 点击行仍打开详情；点击标记按钮仍只更新对应私有记录，且不打开详情。

## 响应式边界

- 宽屏：上述稳定表格布局，表头继续 sticky。
- 中等宽度：筛选字段自动分行，但字段内部不拆开；表格保持列规则，必要时允许横向查看而不压缩成不可读的断词。
- 小屏：继续使用现有卡片列表；个人标记沿用 2 × 2 结构并与卡片标题对齐。

## 中性公开地址

- 当前页面品牌 `AI Anim Rank` 不包含个人名；个人名只出现在 Workers 账户子域名中。
- 发布目标改为 Cloudflare Pages 静态站点，使用 `ai-anim-rank.pages.dev` 形式的免费公开地址，不暴露账户子域名。
- 现有远程同步尚未配置，因此 Pages 发布不改变已启用功能。将来若启用同步 API，可继续由独立 Worker 承载；页面只需更新其 API 配置，不需要将个人进度公开。

## 验收标准

- 1440px 宽桌面下，筛选字段的标签与控件不再错位，四个个人标记呈整齐 2 × 2。
- 常见长标题和长类型不会造成评分、标记列的非预期换行。
- 默认 Light 与 Dark 均可读，连续浏览 300 条记录不受内部滚动容器限制。
- Pages 地址可公开访问，且地址中不包含用户个人名。
- 既有进度联动、备份导入导出、筛选排序、PWA 与加密同步相关测试继续通过。
