# AI Anim Rank：首版设计规格

## 目标

做一个可公开访问、可安装的全球动画作品榜单。任何访问者都能浏览同一份公开排名；每位访问者的观看、评价与偏好数据默认仅属于自己，并且不会被公开显示或上传为明文。

首版需要同时满足：

- 浏览、搜索、筛选和排序一份可信的 Top 300 榜单；
- 标记已看、已评价、推荐与不感兴趣，并自动保持状态规则；
- 不创建账号、不收集邮箱，并支持浏览器间同步；
- 允许用户将私有数据导出为 JSON，作为独立备份；
- 用免费托管能力发布公开应用与小型同步服务。

## 公开榜单与数据治理

### 作品主键与字段

每个作品以内部稳定 `workId` 为主键，映射 AniList、MAL 与 Bangumi 的外部 ID。公开数据包含：排名、规范中文名、英文名或原名、年份、制作公司、题材、综合分及来源评分。

作品颗粒度是单一电视动画、电影、OVA 或 ONA 条目，不把独立续作、总集篇和特别篇自动合并。只有在数据源确实指向同一作品时才合并，避免不同季的《灵能百分百》等条目被错误去重。

### 排名方法

主评分源是 MyAnimeList、Bangumi 和 AniList。各源先按评分分布做标准化，再按预先公开的权重组合；低于各源最低有效评分人数的条目不进入该源的加权计算。综合分与各源分数、抓取时间和方法版本都作为可追溯元数据保存。

可以加入额外来源，但只有同时满足以下条件才会纳入主排名：

1. 具有可稳定、可合法复核的获取方式；
2. 明确区分作品和季，且能可靠映射到内部 `workId`；
3. 评分样本量和更新时间可取得；
4. 能完成历史回测，确认不会系统性重复三大源的同一用户群或扭曲新旧作品。

新增来源先以实验列展示和报告，不直接改变公开排名。排名快照只在数据维护流程运行时更新；访问者浏览站点时不直接请求第三方评分网站。

### 数据质量门槛

发布快照前必须检查：外部 ID 唯一、作品标题重复、系列关系、中文译名、年份/工作室缺失、评分范围和前 300 结果抽样。检查失败时不替换线上快照。不会用模拟或占位数据冒充真实 Top 300。

## 私有进度与交互

### 私有状态

每位用户的进度以 `workId` 键控，包含：

- `watched`：已看；
- `reviewed`：已评价；
- `recommended`：推荐；
- `notInterested`：看过但不感兴趣；
- `note`：仅本人可见的可选备注。

选中“已评价”、“推荐”或“不感兴趣”会自动选中“已看”。“推荐”和“不感兴趣”互斥。任何私有状态都不会进入公开榜单、搜索索引、分享链接或其他用户的界面。

### 榜单工作台

桌面端由搜索、题材/状态筛选、排序控制、个人统计和独立内部滚动榜单构成。榜单表头固定在榜单容器内，不与页面导航重叠。支持按排名、综合评分和年份升降序；状态筛选包括全部、已看、未看、已评价、推荐、不感兴趣和已看未评。

移动端使用可展开的作品行和底部抽屉来完成标记，保留搜索和筛选，避免宽表格导致操作列错位。推荐、未感兴趣等颜色只在当前用户的浏览器内呈现。每次本地保存或同步完成提供简短、非打扰式反馈。

## 本地优先与匿名加密同步

### 默认行为

不启用同步时，私有进度仅保存在浏览器 IndexedDB；用户可随时导出或导入版本化 JSON 备份。导入先校验格式、版本与 `workId`，再由用户确认合并或替换。

### 启用同步

首次启用时，客户端生成高熵的 12 词恢复短语及同步标识。恢复短语派生客户端加密密钥；记录内容在浏览器中加密后才上传。Cloudflare 数据库只保存同步标识、版本信息和密文，不保存作品状态明文、邮箱或用户名。

当前设备将同步凭证留存在本地，因此正常使用无需重复输入，表现为自动同步。新设备通过输入恢复短语或扫描迁移二维码接入同一份密文数据。恢复短语不应由用户自行编造，也不能出现在 URL、分析日志或客服可见记录中。

### 风险与恢复

启用前、恢复短语展示页和同步设置页都必须用清楚的中文提示：恢复短语是唯一凭证；遗失短语且清空所有已接入设备的浏览器数据时不可恢复；泄露短语的人可以解密并修改该私有记录。用户需要明确确认自己已保存短语，才能完成启用。

本地修改先立即完成，再异步同步。冲突以条目级“最后修改时间”合并，并保留更新版本号，避免一个设备的整包旧数据覆盖另一设备的新标记。断网时继续本地工作，网络恢复后重试同步。

## 部署架构

主应用是静态 PWA，包含榜单快照、界面和离线资源，由 Sites 发布为公开访问地址。PWA manifest 和 service worker 仅缓存公开应用资源与当前设备已解密的本地状态，不缓存恢复短语到公开可读文件。

匿名同步由单个 Cloudflare Worker 和 D1 数据库承担。Worker 只处理加密载荷的读取、条件写入和版本冲突，不负责鉴权、账号、分析或第三方数据抓取。免费层足以支撑首版的低频个人同步；若未来流量接近平台限制，优先增加客户端退避、大小限制与滥用防护，而不弱化加密模型。

## 验收与测试

首版必须验证：

- 状态联动和推荐/不感兴趣互斥；
- 筛选、搜索和三种排序正确组合；
- 私有状态在重新打开浏览器后保留，且从公开数据中不可读；
- 加密同步不上传状态明文，新设备可用恢复短语接入；
- 冲突合并、断网重试、导入导出与错误提示；
- 键盘、触屏和窄屏布局可用；
- 数据快照的唯一性、字段完整性与抽样准确性检查。

## 非目标

首版不实现公开评论、社交关注、用户主页、推荐算法、付费服务或从访问者浏览器直接抓取第三方评分网站。不会承诺“绝对权威”；榜单会公开方法与快照版本，让评分口径可质疑、可更新。
